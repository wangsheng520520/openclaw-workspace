**修复**:

## 中文字体修复 (21:04~21:20)

**问题**: 系统无中文字体，Pillow 无法渲染中文，导致封面图文字为空
**尝试**:
- `sudo apt install fonts-wqy-zenhei` → 需密码，失败
- GitHub raw download + CDN download → WSL2 DNS 问题，均失败（55字节伪文件）
- Playwright HTML screenshot → 可行但每次需启动浏览器，慢
**最终方案**: WSL2 可访问 Windows 宿主字体目录 `C:\Windows\Fonts\`
- `ln -sf /mnt/c/Windows/Fonts/simhei.ttf ~/.fonts/SimHei.ttf`（黑体，9.7MB）
- `ln -sf /mnt/c/Windows/Fonts/simsun.ttc ~/.fonts/SimSun.ttc`（宋体，18MB）
- `ln -sf /mnt/c/Windows/Fonts/simkai.ttf ~/.fonts/SimKai.ttf`（楷体，11.8MB）
- `ln -sf /mnt/c/Windows/Fonts/Deng.ttf ~/.fonts/Deng.ttf`（等线，16.3MB）
- `ln -sf /mnt/c/Windows/Fonts/Dengb.ttf ~/.fonts/DengBold.ttf`（等线粗体）
- `ln -sf /mnt/c/Windows/Fonts/HYZhongHeiTi-197.ttf ~/.fonts/HYZhongHeiTi.ttf`（汉仪中黑）
- `fc-cache -fv ~/.fonts/` → 刷新字体缓存
- 验证: `fc-list :lang=zh` → 6 款中文字体，Pillow 渲染测试 ✅
- **教训**: WSL2 下直接 symlink Windows Fonts 目录是中国字体问题的标准解法

### 教训
- **硬编码日期是 Cron 任务的定时炸弹**: 日期过期后文件不存在 → agent 静默跳过 → cron 显示 "ok" 但无产出
- **Cron payload 应该用 `date` 命令动态计算日期，永远不要写死具体日期**
