- 需要 `agents.defaults.bootstrapMaxChars: 25000` 这个受保护字段 → 直接编辑 openclaw.json

**MCP 泄漏根本问题**:
- 版本 v2026.4.24 已内置修复，但 WSL2 + systemd user instance 不完全支持进程组管理 → SIGTERM 杀不死 MCP 子进程（systemctl stop 总是超时被 kill）
- `gateway restart` (SIGUSR1) 完全不杀子进程 → 每次累积
- `systemctl --user restart` 比 SIGUSR1 好但仍残留
- **当前策略**: 主动监控 MCP 进程数，超过 20 个时 `pkill` 清理 + `systemctl restart`

## 关键经验

**心跳架构原则**:
- 心跳必须用独立 session lane，否则每 30 分钟阻塞主会话
- 独立 session 需要完整上下文 (lightContext: false)，否则空消息导致 API 拒绝
- 模型选择：ark-code-latest (主模型) 最快且无限流问题

**Gateway 重启选择**:
- `gateway restart` (SIGUSR1) = 不杀 MCP 子进程，每次累积累积
- `systemctl --user restart` = 部分杀子进程，WSL2 下仍有残留
- 最佳实践：主动监控 MCP 进程数，跨阈值时 pkill + systemctl restart

**受保护字段写入**:
- `bootstrapMaxChars` 等受保护字段不可通过 `gateway config.patch` 修改
- 需直接编辑 `openclaw.json` + `gateway restart` 热重载
