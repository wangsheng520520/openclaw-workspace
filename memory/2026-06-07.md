## Tavily secrets.resolve 失败(22:08~22:20)
- **症状**:飞书 channel 反复发 "secrets.resolve failed: plugins.entries.tavily.config.webSearch.apiKey is unresolved in the active runtime snapshot"
- **频率**:每次 agentTurn 一次(20:20/20:22/20:34/20:40/20:53/20:57/21:02/21:04/21:24/21:27/21:29/21:51/21:56/22:07/22:07+ = 6+ 次)
- **根因**:
  - Tavily bundled plugin 的 `openclaw.plugin.json` 缺 `configContracts.secretInputs.paths` 字段
  - `collectPluginConfigAssignments` 跳过 Tavily 的 apiKey 解析
  - `resolvedConfig.plugins.entries.tavily.config.webSearch.apiKey` 仍是 ref 对象
  - `analyzeCommandSecretAssignmentsFromSnapshot` 检查 → 报 unresolved
- **不报 doctor**:`openclaw doctor` 用 `readJsonPointer` 直接读文件,能解析;只有 runtime 才暴露
- **不影响其他工具**:feishu/models 等其他 secret target 正常
- **修复**:`openclaw secrets reload` — atomic swap runtime snapshot,触发完整 resolve 路径
- **验证**:
  - `openclaw secrets audit` → `unresolved=0` ✅
  - 3 次 agent turn 后 journal 中 0 个 `secrets.resolve` 失败 ✅
- **学到**:
  - 飞书 channel 不只是发 heartbeat,也会转发 runtime warnings(噪音来源)
  - `openclaw secrets reload` 是处理 runtime secrets drift 的标准工具
  - Tavily tool 在 reload 前是 ready 但实际调用失败(因 cfg.apiKey 是 ref)
