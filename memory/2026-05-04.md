## 博客监控 cron 修复 (22:09~22:25)

### 问题
v2026.5.2 升级后博客监控三次投递失败：
- 08:11 `Unsupported channel: feishu`（插件未恢复）
- 20:00 `Outbound not configured`（插件已装但 delivery routing 不通）
- 22:02 同上

### 根因
v2026.5.2 将 feishu 外部化为 npm 插件后，cron delivery `mode: "announce"` 和 `message` 工具的出站路径不通，虽然 WebSocket 入站正常。

### 修复
1. `delivery.mode` 改为 `"none"`（绕过失效的 channel delivery）
2. Prompt 中指示 agent 用 `message` 工具或 `lark-cli im +messages-send` fallback
3. 添加 `message` 工具到 toolsAllow
4. 用户 `systemctl --user restart openclaw-gateway` 后交付恢复系统自动补发了积压的两份报告

### 关键发现
- feishu WebSocket 连接正常（`WebSocket client started` + `bot open_id resolved`）
- `delivery-recovery` 在 Gateway 启动后成功恢复投递（`Recovered delivery ... on feishu`）
- `message` 工具返回 `Channel is unavailable` 是 v2026.5.2 工具层路由问题，非 channel 问题

## 博客监控 cron 修复 (22:09~22:25)

### 升级方式
- git fetch/pull 在 WSL2 下持续超时（GitHub 44MB 仓库，HTTPS 传输挂起）
- 改用 `npm pack @evomap/evolver@1.78.9` → 下载 tarball → 解压覆盖源文件
- 保留本地资产备份后恢复

### 升级结果
| 项目 | 升级前 | 升级后 |
|------|--------|--------|
| 版本 | v1.75.0 | v1.78.9 |
| 相差 | - | 9 个版本 |
| Evolver 运行 | ✅ PID 92578 | ✅ PID 94404 |
| Hub 资产同步 | 本地 | 21 Capsules + 5 Genes 从 Hub 恢复 |

### 关键教训
- **WSL2 git fetch 不可靠**: 耗时>120s 仍被 SIGTERM，建议用 npm pack 替代
- **资产文件** events.jsonl 被 npm tarball 覆盖为 0B（之前 hard rollback 可能已清空）
- **Hub 同步恢复**: `A2A_HUB_URL=https://evomap.ai node index.js sync --scope=all --export=backup.gepx` 成功恢复 21 capsules
