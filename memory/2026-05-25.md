## EvoMap node_secret 注册问题解决 (05-24 21:30 ~ 22:46)

### 问题
Evolver 向 Hub 发送 hello 后，Hub 返回 `"Unknown message type: hello"` 而非 `node_secret`。

### 根因
Evolver 的 hello 消息**自带 sender_id（自生成的 node_9e79fe6d6b15）但无有效 node_secret**。Hub 将其当作"未认证的已有节点重连"，拒绝识别消息类型。

### 正确的首次注册流程（按官方 SKILL.md）
```bash
# 首次注册：不带 sender_id，Hub 自动分配 node_id 和 secret
curl -s -X POST https://evomap.ai/a2a/hello \
  -H "Content-Type: application/json" \
  -d '{
    "protocol": "gep-a2a",
    "protocol_version": "1.0.0",
    "message_type": "hello",
    "timestamp": "<ISO timestamp>",
    "payload": {"capabilities": {}},
    "env_fingerprint": "<fingerprint>"
  }'
```

### 尝试过的节点
| 节点 | 方式 | 结果 |
|------|------|------|
| `node_9e79fe6d6b15` | Evolver 自带 sender_id | ❌ Hub 拒绝 |
| `node_710c5608ca9076bd` | 手动 API 注册（无 sender_id） | ✅ 注册成功，claim_url 未使用 |
| `node_c4dc5cf4ebcfc08a` | 手动 API + rotate_secret | ✅ 注册成功，claim_url 未使用 |
| **`node_9e79fe6d6b15`** | 用户在 evomap.ai 重置 secret → 填入 .env → 重启 | ✅ **最终方案** |

### 最终配置
```
A2A_NODE_ID=node_9e79fe6d6b15
A2A_NODE_SECRET=0d03ceba3ad2a116756e73fdf46b57eb8965948f5f8317d3bfbec04f14656f44
A2A_HUB_URL=https://evomap.ai
```

### 关键教训
1. **首次注册不带 sender_id**：Hub 才会分配新 node_id + secret
2. **已注册节点重连**：需要带 sender_id + 有效 node_secret（在 Authorization header 中）
3. **找回旧节点**：目前只能在 evomap.ai Web 仪表板手动 "Reset Secret"
4. **Evolver daemon 的 hello 带 sender_id**：这是给已有 secret 的节点用的，不适合首次注册
