# 06-13 完美模式固化设计

**日期**: 2026-06-14
**作者**: Ada (with user)
**状态**: 待用户审查
**范围**: 全部通道（飞书私聊 / 飞书群聊 / webchat / CLI / 任何用户输入 turn）
**触发方式**: 所有场景（每条有意义的用户消息）
**实现形式**: 写入 `SOUL.md` / `AGENTS.md`，固化进默认人格

---

## 1. 背景与依据

- **14-17 完美样本**（2026-06-11 14:00-17:00，session `24587d3a-...`）：

  18 个 turn 全部遵循“**收到 — bold + Using [using-superpowers] + 链式 Using**”三元素模板，完美率 18/18 = 100%（仅看技术任务场景）。

- **06-13 样本**（2026-06-13，飞书私聊，40 条消息）：

  13 次有意义的助手响应里 11 次严格遵循相同模板。两份独立证据交叉验证。

- **违反模式**：
  - 续轮 / 单纯状态查询（"继续" / "看一下"）最常违反
  - cron / heartbeat / 系统类 turn 不套此模板（合理）
  - 多次违反时，用户在 23:32 明确点名“你又没有启用使用超能力技能”

- **结论**：14-17 / 06-13 模式有可观察、可复现、可验证的形式，应该被提升为**默认人格行为规范**，与现有 `AGENTS.md` “每条消息先检查技能” 互为补充。

## 2. 设计目标

固化后 **每条**有明确技术任务的用户消息，助手第一句回复**必须**形如：

```text
收到 — **<一句话概括的用户意图>**
Using [using-superpowers](~/.openclaw/workspace/skills/using-superpowers/SKILL.md) to <流程目的>
[可选] Using [<secondary-skill>](path) to <次要目的>
...（其余思考/证据/计划/状态）
```

### 必须满足的不变量

1. **永远**以“收到 —”开头（除非极短澄清）。
2. **永远** 链接 `using-superpowers` SKILL.md 路径。
3. **复杂任务** 链式追加 `Using [...]`（至少 1 个，最多 4 个，避免过长）。
4. **不要** 把 `Using` 当装饰性前缀——`Using` 后必须能直接对应到该技能的方法学目的。

## 3. 适配场景矩阵

| 场景 | 是否套 06-13 模板 | 理由 |
|---|---|---|
| 用户明确技术任务（修复 / 查 / 改） | ✅ 必套 | 14-17 / 06-13 高频场景 |
| 用户纠错 / 质疑 | ✅ 必套 | 含 `receiving-code-review` 链式 |
| 用户说"继续" | ⚠️ 可省 | 续轮易破坏模板；但 06-13 23:41 “继续”之后仍是模板 |
| 短确认（"好" / "OK"） | ❌ 不套 | 极短澄清不强制 |
| 系统 / cron / heartbeat turn | ❌ 不套 | 不经过用户回合 |
| 心跳报告 | ❌ 不套 | `HEARTBEAT_OK` 等 |

判断规则：**如果不确定，就套**。这与 06-13 “宁愿多套” 一致。

## 5. AGENTS.md 变更

### 5.1 改写“强制流程”章节

把现有：

## 6. 不变量自检

1. **占位符扫描**：无 TODO / 模糊
2. **内部一致性**：SOUL.md 与 AGENTS.md 模板一致
3. **范围检查**：本次仅改两个文件，5 分钟级实施
4. **模糊性检查**：
   - "极短确认" 边界：长度 < 8 字符或纯 emoji
   - "系统 turn"：排除所有 cron / heartbeat / SESSION-STATE 内部 turn

## 7. 实施步骤

1. 用户批准本规格
2. 编辑 `SOUL.md`：在“表达风格”前插入新章节，版本号 v2.1 → v2.2
3. 编辑 `AGENTS.md`：替换“强制流程”章节，新增证据 bullet
4. 触发一次心跳 / 短消息自检，确认新模板被加载
5. 30 分钟后看行为样本，确认遵守率 ≥ 90%

## 8. 风险与回滚

- **风险 1**：模板过严，导致短响应被迫变长 → 缓解：保留“极短确认 / 系统 turn” 例外
- **风险 2**：技能链接过期 → 缓解：技能路径用 `~/.openclaw/workspace/skills/<name>/SKILL.md` 相对路径
- **回滚**：用户不喜欢时，恢复 SOUL.md / AGENTS.md 即可（git diff 可视化）

## 9. 验证方法

- 24 小时内统计助手响应的“Using [using-superpowers]” 出现率
- 目标：技术任务 turn ≥ 90%
- 抽样 5 个 turn 人工 review
- 不达标 → 调整例外规则
