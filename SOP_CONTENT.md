# SOP — 内容生产工作流

> "收集-整理-创作-分享-沉淀" 的自动化闭环

---

## 阶段一：信息收集 (01-灵感库)

**触发**：用户发送链接/文章/想法

**流程**：
1. 解析内容（网页用 web_fetch，文档用 document-pro）
2. 提取核心信息
3. 使用 **AI采集模板** 创建笔记
4. 存入 `01-灵感库/` 目录
5. 设置 `stage: 灵感`

**命令示例**：
```bash
# 创建灵感笔记
obsidian-cli create "01-灵感库/主题名" --content "..."
```

---

## 阶段二：选题策划 (02-选题池)

**触发**：用户说"整理选题"、"规划内容"

**流程**：
1. 扫描 `01-灵感库/` 中的笔记
2. 按主题聚合，识别热点和趋势
3. 筛选有价值的选题
4. 移动到 `02-选题池/`
5. 更新 `stage: 选题`

**命令示例**：
```bash
# 移动笔记（自动更新链接）
obsidian-cli move "01-灵感库/主题名" "02-选题池/主题名"
```

---

## 阶段三：内容创作 (03-内容工厂)

**触发**：用户说"开始写作"、"创建内容"

**流程**：
1. 从 `02-选题池/` 选取选题
2. 检索 Obsidian 中相关笔记（search-content）
3. 构建大纲
4. 撰写初稿
5. 移动到 `03-内容工厂/`
6. 更新 `stage: 创作`

---

## 阶段四：发布归档 (04-已发布归档)

**触发**：用户确认发布

**流程**：
1. 最终审校
2. 适配目标平台格式
3. 移动到 `04-已发布归档/`
4. 更新 `stage: 已发布`
5. 添加发布日期和平台信息

---

## 安全规则（不可违反）

1. 🚫 **绝不使用** mv/rm/cp 操作 Vault 文件
2. ✅ **必须使用** obsidian-cli move/create
3. ✅ **必须注入** YAML Frontmatter
4. ✅ **操作前** 先用 search-content 确认目标
5. ✅ **移动后** 验证链接完整性

---

```
├── 03-内容工厂/       ← stage: 创作
├── 04-已发布归档/     ← stage: 已发布

```
